Loading… Install npm In order to get started, npm must be installed. To install it, follow the instructions at the package managers page. Using npm Angular 2 and Webpack are both based on npm. Any changes to TypeScript or CSS files will need to be re-webpacked by running npm run build. The run build argument is taken from scripts section of the package.json file: { "scripts": { "build": "rimraf public_html/dist && webpack --config webpack/webpack.live.js --progress --profile --bail" } } The scripts section contains commands that need to be run again and again in a project. If your project has another repetitive task, another script may be added to the scripts section. The build script chains three commands run in sequence: rimraf public_html/dist: clean out the frontend facing webpack generated directory webpack --config webpack/webpack.live.js --progress --profile --bail: run webpack with the live configuration and build the Angular 4 app Starting The Dev Server The Dev Server is used to create a environment that allows users to test their Angular App in an efficient manner Startup the terminal. Running the Build Remotely Running the build remotely is the final step in deploying an Angular app. The build will then be reflected on the server and will be what the end users will tangibly see. It is highly recommended the build first, so one can eliminate common errors first. To remotely build the app: Connect to the server using SSH. cd to the project directory Type npm run build Once the remote build is complete, the site is immediately deployed and ready. Reload the site and ensure it is working as expected.
